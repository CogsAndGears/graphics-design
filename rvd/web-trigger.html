Web Trigger allows starting your application using a simple HTTP call

Web Trigger URLNote that from and to URL parameters can be omitted if fallback values have been defined. Also, additional parameters can be appended to the url. They will be available as module scoped RVD variables when the application starts

https://cloud.restcomm.com:443/restcomm-rvd/services/apps/AP312bf0a27d4348c0aa77aade18df7ab3/start?
from=12345&to=+1231231231&token=mysecret

Access TokenProtect your Web Trigger URL with an access token. If set, make sure you provide the same token parameter when making the HTTP call. Leave it blank to allow free access.

Access token is empty! Your trigger may only be accessed using HTTP basic authentication.

Fallback ParametersYou can optionally provide fallback To and From values in case they are not supplied in the HTTP call.

Note that From and To values use the Restcomm RCML dial syntax. For example: +1232343294 or client:bob or conference:1234 or sip:+12839283849@server.com
